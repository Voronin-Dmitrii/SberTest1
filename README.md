## Задание №2. Прогнозирование взносов и выдач

Задача — построить прогноз сумм **внесений (`cashin`)** и **выдач (`cashout`)** по каждому устройству самообслуживания (банкомату) на **период с 01.01.2025 по 31.01.2025**, используя исторические данные за **2024 год**.

Данные предоставлены в виде CSV-файла `test_task_1.csv`:

| Поле     | Описание |
|-----------|-----------|
| `atm_id`  | Уникальный идентификатор банкомата |
| `date_at` | Дата операции |
| `cashin`  | Сумма внесённых средств |
| `cashout` | Сумма выданных средств |

Особенности данных:
- возможны пропуски по датам (не каждый день есть записи);
- 10 уникальных устройств (`atm_id`);
- суммы варьируются по устройствам и дням.

---

## Логика модели Prophet

Модель Prophet строит прогноз на основе трёх компонентов:

- **Trend** — общий тренд (рост/спад активности);
- **Seasonality** — недельная и годовая сезонность (например, пики в выходные);
- **Holidays** — влияние праздничных дней, автоматически учитываемое для России (`add_country_holidays('RU')`).

В отличие от классических моделей (ARIMA, Holt-Winters), Prophet:

- не требует ручной настройки параметров и подбора порядков моделей;
- устойчив к неполным данным и выбросам;
- есть встроенные country_holidays
